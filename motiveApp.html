Motive App - HackNC

  • GitHub

During the 2020 HackNC hackathon, I worked with a team of students from UNC Chapel Hill, UC Irvine, and UC Berkely to create a task management app. We found that during the pandemic, it became increasingly difficult to motive yourself to get even those basic tasks done like doing the dishes or that homework assignment due in 3 hours. Our solution to this is Motive App, a task management app that acts more like a game than a management app.

Created with a Java backend and Java Swing frontend, Motive App takes tasks that a user has entered and assigns them point values based on the size and scope of the task. For example, something like doing the dishes would be assigned 5 points whereas something like studing for an exam is a larger task and would be assigned 20 points. Upon completion of a task, those points are awarded towards the user and is counted towards their daily point goal. By setting a daily point goal, it helps the user slowly regain their motivation and encourage them to take slow steps to get the things they need to get done.

Motive App Features

Features include:

  • Matlab simulation
  • 2D array
  • Variable "random walks" to show how increased sample sizes improves route length

Motive App Future Improvements

Future Improvements include:

  • Machine Learning to automatically assign point values to tasks
  • Machine Learning to automatically assign tasks that a person repeatedly does on a daily or weekly basis
  • Better UI
  • Reward system where the user can redeem the points for rewards

At first, tackling something as complex as ant behavior seemed a very daunting task. However, I found that once we broke it down and made some simplifying assumptions it became much easier to create a model for. Although this model for ant pathfinding behavior is by no means perfect, it shows how you can easily create something that models the natural world pretty well. Even as a simple model, there are so many potential applications for this type of program ranging from the Traveling Salesman problem to identifying the shortest route between two points on a map.
